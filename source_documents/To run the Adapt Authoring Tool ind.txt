To run the Adapt Authoring Tool indefinitely, you can use a process manager like pm2. Here's how you can install pm2 and use it to run the Adapt Authoring Tool:

Install pm2 by running the following command:

Copy code
sudo npm install -g pm2
Navigate to the directory where the Adapt Authoring Tool is installed. For example:

bash
Copy code
cd /opt/adapt_authoring
Start the Adapt Authoring Tool using pm2 by running the following command:

sql
Copy code
pm2 start npm -- start
This will start the Adapt Authoring Tool as a background process managed by pm2.

Save the pm2 configuration so that it will automatically start the Adapt Authoring Tool on boot:

Copy code
pm2 save
This will save the current pm2 configuration so that it will automatically start the Adapt Authoring Tool when your instance is rebooted.

You can now access the Adapt Authoring Tool by navigating to your instance's IP address on port 5000. The Adapt Authoring Tool should continue to run even if you disconnect from your instance or log out of your SSH session. If you need to stop the Adapt Authoring Tool, you can do so by running the following command:

vbnet
Copy code
pm2 stop npm

                $ pm2 monitor

                $ pm2 startup
